Detailed Description

Helper class used for calls to Message().

Member Function Documentation

virtual void IM2MMessageListener::MessageHelper::AddPrivateData (PrivateData * data)
pure virtual

This methods allows the Listener to specify a pointer to private data that will be passed into subsequent calls to the Message() and Validate() methods.

Note that the M2MMessageManager will only call Validate() must provide private datarequired* in order to get called during Validate().

Although the data is private, it's lifetime is controlled by the M2MMessageManager and will be deleted by it at the end of the 'transaction'.

virtual void IM2MMessageListener::MessageHelper::DeferMessageToEnd ()
pure virtual

This method allows the Listener to have the message re-delivered after all other Listeners of the same message have been called. See GetWasDeferred() above for more information.

virtual bool16 IM2MMessageListener::MessageHelper::GetFromPreChange () const
pure virtual

Returns kTrue if the original source of the message was from a call to any of the M2MMessageManager::PreChangeMessage() or M2MMessageManager::PreChangeMessageAllowDuplicate() methods. Messages of this type come from a call to PreDirty() and thus mean that the changing implementation has not yet been changed. Callees are free to query the changing implementation for pre-changed values.

Returns
kTrue if from prechange, kFalse otherwise
virtual bool16 IM2MMessageListener::MessageHelper::GetIsDuplicateChange () const
pure virtual

In the most common usage driven from PMPersist::PreDirty() the M2MMessageManager only delivers the same message to the same Listener once per "transaction" that is ended by a call to Validate().

Certain implementations however, such as IHierarchy and ITransform, affect other bosses other than themselves and so repeat changes are meaningful to Listeners.

We expect that the number of implementations in this class to be very, very small (otherwise performance will suffer).

Returns kTrue if the original source of the message was from a call to any of the M2MMessageManager::PreChangeMessageAllowDuplicate() methods AND the message is a duplicate of a message that was already received by the target of the message.

Returns
kTrue if a duplicate message.

virtual bool16 IM2MMessageListener::MessageHelper::GetWasDeferred () const
pure virtual

Messages are delivered to the set of Listeners listening to a boss in an indeterminate order. Some Listeners, because the data they are managing is itself changed through the actions of other Listeners, have a difficult time ensuring cached data is properly invalidated. The kPageItemTransformImpl is good example of this.

The DeferMesageToEnd() method provides a way to affect this delivery order in a limited way by asking that the message be re-delivered to the Listener after all the other Listeners have been called. This works when the the actions of the Listener itself have no side effects on other bosses or implementations.

Note that the callig order between Listeners who have also asked to have the message deferred to the end is still indeterminate

Returns
kTrue if the message was deferred
