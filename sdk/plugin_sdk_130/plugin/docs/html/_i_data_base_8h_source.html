681  // More UID functions. Move up with rest between builds; but don't change vtable now [][]
682  //------------------------------------------------------------------------------------
683  // Creates a new persistent object in the database. The caller specifies the desired
684  // UID of the new object. This will succeed under the following conditions:
685  // 1. The requested UID is in the valid range - (currently: 1 <= UID <= 2^31-1)
686  // 2. The UID has either never been used, or can be re-used
687  // A UID that was used before can only be re-used provided that it does not exist
688  // in either the temporary file or the user file. This requires at least calling
689  // DeleteUID and DestoryUID, and performing a save operation afterwards if the
690  // database protection level is greater than kNoProtection.
691  //
692  // There may be three outcomes when calling this function. Callers should be prepared:
693  //
694  // If the function succeeds, the object is created and the function returns kTrue
695  //
696  // If the function fails because the UID isn't available to be used (outside of the
697  // valid range, not available for re-use yet), the function returns kFalse but the
698  // database error state is not set
699  //
700  // If the function fails for some more insidious reason, it returns kFalse and the
701  // database error state is set appropriately
703  virtual bool16 PlacementNewUID(ClassID clsID, UID id) = 0;
