Detailed Description

  • How to acquire one or all tags (kXMLTagBoss)in a document (kDocBoss)
  • How to create, and apply, a style to tag mapping (IXMLStyleToTagMap)
  • How to create, and apply, a style to tag mapping (IXMLStyleToTagMap)
  • How to create a tag
  • How to load tags from an existing tags file
  • How to save the set of tags
  • How to unmap styles from tags
  • How to unmap tags from styles

Exercising

AcquireTag

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option AcquireTag.
  4. If all is well, there should be no asserts (if in debug build).
  5. You should see information about all tags being listed, and an attempt to acquire one by name

CreateAndApplyStyleToTagMap

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option CreateAndApplyStyleToTagMap.
  4. If all is well, there should be no asserts (if in debug build).
  5. You should see some new para styles and char styles were created (Paragraph Styles and Character Styles panels)
  6. You should see some new tags were created, via the Tags panel
  7. You should see a page item created, with styled text. The styled text should be tagged systematically.
  8. Try scrolling down through the text and seeing how the style changes, and note associated tag. Story editor view is useful for this.

CreateAndApplyTagToStyleMap

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option CreateAndApplyTagToStyleMap.
  4. If all is well, there should be no asserts (if in debug build).
  5. You should see new info when you open the Map Tags to Styles dialog. This is available from structure-view popout

CreateTags

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option CreateTags.
  4. If all is well, there should be no asserts (if in debug build).
  5. Open or inspect Tags panel.
  6. You should see some new tags have been created.

LoadTags

  1. Be sure you have run SaveTags first.
  2. Create a new document
  3. Run the snippet ManipulateXMLTags
  4. Choose the option LoadTags.
  5. If all is well, there should be no asserts (if in debug build).
  6. You should see some tags have been loaded in.

SaveTags

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option SaveTags.
  4. If all is well, there should be no asserts (if in debug build).
  5. You should see a message about a file being created.
  6. Save snippet runner log, grab the path to the file and inspect in XML or text editor if you wish

UnmapStylesFromTags

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option UnmapStylesFromTags.
  4. If all is well, there should be no asserts (if in debug build).
  5. Inspect the Map Styles To Tags dialog. This is available from structure-view popout. All mappings should be gone.

UnmapTagsFromStyles

  1. Create a new document
  2. Run the snippet ManipulateXMLTags
  3. Choose the option UnmapTagsFromStyles.
  4. If all is well, there should be no asserts (if in debug build).
  5. Inspect the Map Tags to Styles dialog. This is available from structure-view popout. All mappings should be gone.
